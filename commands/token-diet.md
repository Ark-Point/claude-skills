# /token-diet — 주간 토큰 ROI 점검

매주 실행. 토큰을 덜 쓰는 게 아니라 **잘 쓰고 있는지** 진단한다.

## 철학

- 도구가 많은 건 capability. 안 쓰는 도구가 많은 게 문제.
- 좋은 모델이 기본. 단순한 작업만 싸게.
- "얼마나 쓰는가"보다 "얼마나 잘 쓰는가".

|------|------|------|

| 개발자당 평균 비용 | $6/일, $100-200/월 | Anthropic 공식 |

## Parse Arguments

## WHEN TRIGGERED - EXECUTE IMMEDIATELY

인자에 따라 해당 Phase 실행. 기본값은 `full`.

## Phase 1: 환경 스캔 (병렬 수집)

### 1-A. CLAUDE.md 분석 (실측 기반)

**실행**:
1. Read로 글로벌 CLAUDE.md 읽기. `@` 줄에서 참조 파일 경로 추출.
2. Bash로 각 파일의 바이트 수 측정: `wc -c < {파일경로}`
3. 토큰 추정: **바이트 수 ÷ 5** (한영 혼용 기준)

수집: 글로벌/프로젝트 CLAUDE.md, @참조 파일 — 각 줄 수 + 바이트 수 → 토큰 추정

### 1-B. MCP 서버 현황

**실행**:
1. Read로 `~/.claude/settings.json` mcpServers + `.mcp.json` 확인
2. Bash로 `claude mcp list 2>/dev/null` → 활성 서버/도구 수
3. 연결 안 되는(미인증/에러) 서버 식별

수집: 서버별 상태, 도구 수, Tool Search 적용 여부

### 1-C. 스킬/플러그인 + 유령 스킬 탐지

**실행**:
1. Glob으로 `~/.claude/commands/`, 플러그인 스킬 파일 카운트
2. 세션 데이터에서 최근 14일 스킬 호출 이력 확인:
   - Bash: `grep -r "skill" ~/.claude/projects/*/session* 2>/dev/null | grep -i "invoke\|trigger\|skill:" | tail -50`
   - 또는 세션 system-reminder의 스킬 목록과 실제 사용 기록 대조
3. **유령 스킬 판정**: 14일간 1회도 호출되지 않은 스킬 목록 생성

> 유령 스킬 = 존재하지만 안 쓰는 스킬. 매 턴 ~100토큰씩 메타데이터 비용만 발생.
> 단, **호출 이력 추적이 기술적으로 불완전할 수 있음** — 확인 불가 시 플러그인 단위로 추정.

### 1-D. 메모리 파일 현황

Bash로 `wc -c` + `wc -l` 측정: MEMORY.md, error-notes.md, @참조 메모리 파일

### 1-E. 모델 & 라우팅

수집:
- 현재 세션 모델 (Opus/Sonnet)
- effort 레벨
- 모델 라우팅 판단: 현재 작업이 Opus급인지, Sonnet으로 충분한지

```markdown

- [ ] 모델 전환: 단순 작업에서 Sonnet/Haiku 활용했는가?
```

### 종합 점수 기준

점수는 "얼마나 잘 쓰는가"를 반영한다.

| 점수 | 기준 |
|------|------|
| **A** | 유령 스킬 0, 미연결 MCP 0, 모델 라우팅 적절, CLAUDE.md 경량 |
| **B** | 유령 스킬 5개 이하, 미연결 MCP 0, 모델 라우팅 대체로 적절 |
| **C** | 유령 스킬 10개 이하, 또는 미연결 MCP 1개 |
| **D** | 유령 스킬 10개+, 또는 미연결 MCP 2개+, 또는 CLAUDE.md 비대 |
| **F** | 위 문제 3개+ 복합 |

> 스킬이 많아도 **쓰고 있으면 OK**. 안 쓰는데 있는 게 문제.
> Opus 사용은 감점 아님. 단순 작업에 Opus만 고집하면 주의.

## Phase 3: 자동 수정 제안 (fix 모드)

`fix` 인자일 때만 실행. 각 수정 전 **사용자 승인 필수**.

### 수정 가능 항목

| 문제 | 제안 |
|------|------|
| 유령 스킬 (14일 미사용) | 플러그인 비활성화 또는 command 파일 삭제 제안 |
| 미연결 MCP 서버 | disable 제안 (/mcp에서 사용자가 직접) |
| CLAUDE.md 200줄 초과 | 코드에서 유추 가능한 규칙 → 삭제 후보 |
| MEMORY.md 200줄 초과 | 오래된/중복 엔트리 → 삭제 후보 |
| error-notes.md 15줄 초과 | 가장 오래된 엔트리 → 삭제 후보 |

**수정하지 않는 항목**:
- MCP disconnect (사용자가 /mcp에서 직접)
- CLAUDE.md 핵심 규칙 (NEVER/ALWAYS)
- 최근 1주 메모리 엔트리

## 원칙

1. **도구는 capability** — 쓰는 도구는 건드리지 않는다
2. **유령만 잡는다** — 14일 미사용이 삭제 기준
3. **좋은 모델이 기본** — Opus 사용은 감점 아님. 단순 작업에만 Sonnet 권장
4. **수치 기반** — 줄 수, 토큰 수, 비율로 표현
5. **5분 이내** — 불필요한 딥 스캔 금지
6. **진단만, 실행은 승인 후**
